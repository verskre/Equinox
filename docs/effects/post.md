### 2. Blur

<b>Complexity</b> `menu` - Kernel chunk size of Optical Flow.
1x1 = Highest Quality
4x4 = Fastest<br>
<b>Quality</b> `menu` - Optical Flow's built in quality for Optimization. <br>
<b>Gain</b> `float` - Optical Flow's gain. Can be increased to make more extreme effects, but increasing Shutter Speed Length / Motion Blur Amount is recommended first.<br>
<b>Optical Flow Threshold</b> `float` - Helps filter out noise and artifacts from Optical Flow.<br>
<b>Pre-Shrink</b> `float` - Pre-Shrinks the resolution before Optical Flow TOP. (Very effective for optimization!)<br>
<b>Shutter Speed Frame Rate</b> `menu` - Current Perform FPS: Uses Perform FPS to help match dynamic FPS when rendering to Movie File Out TOPs. 
Project FPS: Matches Project Framerate
Manual FPS: Set the frames per second manually<br>
<b>Manual FPS</b> `int` - Manual override of framerate for pseudo "FPS".<br>
<b>Shutter Speed Length (s)</b> `float` - Affects the length / intensity of the blur. 

Technically: 1 = 360 degree open shutter. <br>
<b>Samples</b> `int` - Affects the amount of pseudo frames that are samples and increases quality of the blur's complexity / smoothness.<br>
<b>Gradient Magnitude (Blur Steepness)</b> `float` - Controls the steepness of the falloff curve at the head and tail of motion blur streaks.<br>
<b>Smooth Vector</b> `toggle` - Controlled by macro. Enables blurring of vector map, which can decrease artifacts at the cost of details. <br>
<b>Pre-Shrink</b> `int` - Vector smoothing pre-shrink.<br>
<b>Filter Size</b> `int` - Vector smoothing filter-size.<br>
<b>Refine Vector Edge</b> `toggle` - Enables vector edge refinement which allows you to refine the smoothing from earlier. This is used to trim the blur back to reduce the blur from spilling onto the background / other GEOs. <br>
<b>Initial Blur</b> `int` - Overall initial blur. 

(Smoothens out the entire shape.)<br>
<b>Threshold</b> `float` - Threshold that trims back the initial blur.

[Trims edges off of initial blur.]<br>
<b>Soften</b> `float` - Softens the edges of the threshold. If set too high, it will appear as YELLOW in the Vector Edge debug pass, which means that it is decreasing some of your blur. If set correctly, it will show as GREEN. 

[Smoothens out edges.]<br>
<b>Post Blur</b> `int` - Blurs the entire edge refinmeent pass to reduce sharp edges on vector map after the refinement blur>thresh process. Appears as RED in the Vector Edge debug pass.  

[Smoothens out final pass.]<br>
